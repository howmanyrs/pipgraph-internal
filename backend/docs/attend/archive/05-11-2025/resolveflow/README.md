## Обзор Проекта

Этот каталог содержит полную документацию для проекта **ранней PARA-классификации заметок**. Вместо того чтобы определять PARA-типы (Project, Area, Resource, Archive) на этапе `extract_nodes` вместе с другими сущностями, мы:

1. **Классифицируем заметку целиком** до создания `EpisodicNode`
2. **Присваиваем PARA label** эпизоду в момент создания
3. **Убираем PARA из entity_types** при вызове `extract_nodes`
4. **Обогащаем связи** через расширенный `edge_type_map` и PARA-специфичные промпты

## Ключевые Решения (TL;DR)

1. **Ранняя классификация**: Классифицируем заметку **до** `extract_nodes`
2. **PARA label на EpisodicNode**: Используем `labels` поле для типа
3. **Убираем PARA из entity_types**: Уменьшаем контекст LLM
4. **Расширенный edge_type_map**: 30+ типов связей вместо 5
5. **PARA-контекст для edges**: Кастомные промпты на основе типа заметки
6. **Обратная совместимость**: Флаг `enable_early_para_classification`
7. **Lazy migration**: Не требуем миграцию всех старых заметок
