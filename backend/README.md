
# PipGraph Backend

**PipGraph** — это бэкенд-движок, который превращает ваши неструктурированные Markdown-заметки (например, из Obsidian) в структурированный граф знаний (Neo4j).

Система действует как "цифровой библиотекарь": она читает ваши заметки, понимает их контекст, предлагает связи с существующими проектами и извлекает важные сущности (задачи, персоны, технологии), не меняя при этом оригинальный текст ваших файлов.

## 🚀 Философия проекта

1.  **Non-Destructive Processing (Неразрушающая обработка)**: Текст ваших заметок неприкосновенен. Система никогда не переписывает тело заметки. Все метаданные сохраняются либо в YAML Frontmatter, либо во внешней базе данных.
2.  **Human-in-the-Loop (Человек в контуре)**: ИИ не принимает решений за вас, а выступает в роли советника.
    *   Если уверенность высока (>95%) — связь создается автоматически.
    *   Если есть сомнения — система создает "Предложение" (Suggestion), которое попадает в ваш Inbox для подтверждения или отклонения.
3.  **PARA Methodology**: Архитектура построена на методологии Тьяго Форте. Каждая заметка классифицируется как часть Проекта, Области ответственности, Ресурса или Архива.

## ⚙️ Как это работает (Pipeline)

Обработка заметки проходит через конвейер на базе **LangGraph**:

1.  **L1 Classification**: Анализ типа заметки и определение контекста PARA (например, "Это заметка о встрече по Проекту Х").
2.  **L2 Proposal Generation**: ИИ ищет связи с существующими узлами графа. Генерируются гипотезы: "Эта заметка относится к проекту Alpha".
3.  **User Decision**:
    *   Пользователь подтверждает или отклоняет предложенные связи через API.
    *   **Cascade Effect**: Подтверждение одной связи может автоматически разрешить похожие предложения в других заметках.
4.  **L3 Entity Extraction**: После подтверждения контекста (например, "Проект Alpha"), система извлекает из текста гранулярные сущности (Задачи, Даты, Технологии), используя этот контекст для повышения точности.

## 🛠 Технологический стек

*   **Core**: Python 3.12+, FastAPI
*   **Orchestration**: LangGraph (State Machine)
*   **Database**: Neo4j (Graph DB)
*   **AI/LLM**: Graphiti, OpenRouter/Cloud.ru (Qwen/GPT-4o)
*   **Validation**: Pydantic

## 📂 Структура

Проект следует строгой слоистой архитектуре:
*   `app/api` — REST интерфейс (без бизнес-логики).
*   `app/workflows` — Оркестрация процессов и управление состоянием.
*   `app/services` — Бизнес-логика и взаимодействие с ИИ.
*   `app/crud` — Атомарные операции с Neo4j.
