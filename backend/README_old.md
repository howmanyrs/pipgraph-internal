
# PipGraph Backend

**PipGraph** — это бэкенд-движок, который превращает ваши неструктурированные Markdown-заметки (например, из Obsidian) в структурированный граф знаний (Neo4j).

Система действует как "цифровой библиотекарь": она читает ваши заметки, понимает их контекст, предлагает связи с существующими проектами и извлекает важные сущности (задачи, персоны, технологии), не меняя при этом оригинальный текст ваших файлов.

## 🚀 Философия проекта

1.  **Non-Destructive Processing (Неразрушающая обработка)**: Текст ваших заметок неприкосновенен. Система никогда не переписывает тело заметки. Все метаданные сохраняются либо в YAML Frontmatter, либо во внешней базе данных.
2.  **Graph-First Structure**: Информация хранится как узлы (Episodes, Entities, PARA Containers) и рёбра (Relationships), что позволяет выполнять сложные семантические запросы.
3.  **PARA Methodology**: Архитектура построена на методологии Тьяго Форте. Каждая заметка может быть связана с Проектом, Областью ответственности, Ресурсом или Архивом.

## ⚙️ Как это работает

Система предоставляет прямой доступ к обработке через REST API:

1.  **Ingestion**: Заметка принимается как "Episode" (событие во времени).
2.  **Entity Extraction**: Система извлекает сущности (задачи, персоны, концепции) из текста с помощью LLM.
3.  **Graph Storage**: Извлечённые данные сохраняются в Neo4j в виде узлов и связей.
4.  **Hybrid Search**: Для поиска релевантных PARA-сущностей используется гибридный поиск (BM25 + векторное сходство).

## 🛠 Технологический стек

*   **Core**: Python 3.12+, FastAPI
*   **Database**: Neo4j (Graph DB)
*   **AI/LLM**: Graphiti, OpenRouter/Cloud.ru (Qwen/GPT-4o)
*   **Validation**: Pydantic
*   **Package Manager**: uv

## 📂 Структура

Проект следует строгой слоистой архитектуре:
*   `app/api` — REST endpoints (см. `/api/v1/dev/*`)
*   `app/services` — Бизнес-логика и взаимодействие с ИИ (PipGraphManager, Graphiti)
*   `app/crud` — Атомарные операции с Neo4j (RelationshipCRUD, EntityCRUD)
*   `app/models` — Pydantic модели для валидации

## 📡 API Endpoints

Все функции доступны через `/api/v1/dev`:
*   `POST /dev/process-note` — Полная обработка заметки с LLM
*   `GET /dev/episodic` — Получение episodic узла по пути
*   `GET /dev/episodic/list` — Список всех episodic узлов
*   `POST /dev/para-entity` — Создание PARA-сущности
*   `GET /dev/para-entity/list` — Список PARA-сущностей
*   `POST /dev/make-suggestions` — Гибридный поиск релевантных сущностей
*   `POST /dev/link-entity-episode` — Создание связи между сущностью и заметкой

См. `app/api/endpoints/dev.py` для полного списка.
